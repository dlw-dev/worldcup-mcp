# World Cup 2026 MCP

MCP server for the **2026 FIFA World Cup** — schedules and results — running on
**Cloudflare Workers**, with match data that **auto-updates via git**.

Ask in natural language (Claude, ChatGPT, Cursor, …): today's matches, kickoff
times, scores, group standings, the knockout bracket, teams and stadiums.

> Built on [`jordanlyall/wc26-mcp`](https://github.com/jordanlyall/wc26-mcp)
> (MIT), ported from a stdio package to a Cloudflare Worker and given an
> automatic data-refresh pipeline (the static npm package didn't have one).

## How it works

- **Worker** (`src/index.js`) exposes a JSON-RPC 2.0 MCP endpoint
  (`initialize` / `tools/list` / `tools/call`) at `POST /worldcup`, `/`, `/mcp`.
- **Tools** (`src/tools.js`): `matches`, `schedule`, `results`, `groups`,
  `standings`, `bracket`, `teams`, `venues`.
- **Data** (`data/tournament.json`) is bundled into the Worker and committed to
  git. It's regenerated by `scripts/refresh-data.mjs` from:
  - [openfootball/worldcup.json](https://github.com/openfootball/worldcup.json)
    (Public Domain) — fixtures, groups, knockout bracket, venues;
  - [football-data.org](https://www.football-data.org/) free tier (optional,
    via `FOOTBALL_DATA_TOKEN`) — live/finished scores, merged by date + teams.

### "Git automático" (auto-update)

`.github/workflows/refresh.yml` runs every 10 min (and on demand): it runs the
refresh script, and **if the data changed**, commits `data/` and **redeploys the
Worker**. So schedules/scores stay current without manual republishing. Live
in-match scores are therefore ~5–15 min behind (GitHub Actions cron cadence); a
Cloudflare Cron Trigger + KV can be added later for sub-minute updates.

## Develop / deploy

```bash
npm install
npm run refresh          # regenerate data/tournament.json
npm run dev              # wrangler dev (local)
npm run deploy           # wrangler deploy → mcp-worldcup.<account>.workers.dev
```

### Secrets (GitHub repo → Settings → Secrets)

- `CLOUDFLARE_API_TOKEN`, `CLOUDFLARE_ACCOUNT_ID` — for `wrangler deploy` in CI.
- `FOOTBALL_DATA_TOKEN` — optional free football-data.org token for live scores.
